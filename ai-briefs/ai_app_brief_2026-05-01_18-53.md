[Introducing Advanced Account Security](https://openai.com/index/advanced-account-security)

### OpenAI：GPT-5 的 Goblin 问题复盘
[Where the goblins came from](https://openai.com/index/where-the-goblins-came-from)  
官方梳理了 GPT-5 中“戈布林”行为异常的时间线、根因和修复方案。**为什么值得关注**：这不是 Bug，是模型个性设计带来的副作用。如果你在个性化助手或游戏 NPC 中用到性格控制，这个复盘有实际参考价值。

### OpenAI：星门（Stargate）算力基建
[Building the compute infrastructure for the Intelligence Age](https://openai.com/index/building-the-compute-infrastructure-for-the-intelligence-age)  
OpenAI 宣布扩容 Stargate 数据中心，支撑 AGI 计算需求。**为什么值得关注**：算力是 AI 应用的“二房东”，扩容直接影响未来 API 响应速度和定价。做规模化落地的人需要留意。

### OpenAI：AI 时代网络安全五步计划
[Cybersecurity in the Intelligence Age](https://openai.com/index/cybersecurity-in-the-intelligence-age)

### Hugging Face：AI 评估正成为新计算瓶颈
[AI evals are becoming the new compute bottleneck](https://huggingface.co/blog/evaleval/eval-costs-bottleneck)  
评估大模型越来越贵，Hugging Face 呼吁业界重视评估成本。**为什么值得关注**：以前大家只关心训练成本，现在评估一个模型可能比训练更耗时。选模型时预算要算上 eval 开销。

### IBM 发布 Granite 4.1 LLM

## 新产品 / 新工具

### Marx Finance：AI 代理辩论市场
[Marx Finance (Product Hunt)](https://www.producthunt.com/products/marx-finance)

### Buda：用代理人组建公司团队
[Buda (Product Hunt)](https://www.producthunt.com/products/buda-2)  
招募 AI 代理人组成同步团队替你运营公司。**为什么值得关注**：一套“虚拟员工”方案，适合初创团队快速搭建客服、运营等自动化流水线。

### HiveTerm：统一 Claude/Codex/Gemini 的工作台
[HiveTerm (Product Hunt)](https://www.producthunt.com/products/hiveterm)  
一个工作台整合 Claude、Codex、Gemini 和你的技术栈。**为什么值得关注**：如果你同时使用多个模型写代码和做 chat，这个工具省去来回切换，直接提升工程效率。

### Postiz：AI 社交排程代理
[Postiz (Product Hunt)](https://www.producthunt.com/products/postiz)  
专为 OpenClaw 等 Agent 设计的社交媒体定时发布器。**为什么值得关注**：内容运营自动化，尤其适合 AI 生成内容后自动分发，降低人工排期成本。

### nudge：AI 自动排程
[nudge (Product Hunt)](https://www.producthunt.com/products/nudge-26)  
丢任务进去，AI 自动排好整周日程。**为什么值得关注**：面向个人时间管理，和 Calendly 等工具不同的是它用 Agent 做主动规划，是个人助理方向的新尝试。

### Genspark for Word：在 Word 里用 AI 写作
[Genspark for Word (Product Hunt)](https://www.producthunt.com/products/genspark)  
在 Microsoft Word 内实现 AI 起草、编辑和调研。**为什么值得关注**：办公场景最直接的产品，如果你是 Microsoft 365 用户，这个插件可无缝替代原有 Copilot。

## 开源项目 / 工程落地

### nanobot：超轻量个人 AI Agent
[github.com/HKUDS/nanobot](https://github.com/HKUDS/nanobot) (⭐41,423)  
“🐈 nanobot：超轻量个人 AI Agent”。**为什么值得关注**：7 天拿 4 万星，主打极轻量级本地运行，可部署在树莓派等边缘设备，适合做离线个人助手或智能家居核心。

### LangGraph：构建弹性语言 Agent 图
[github.com/langchain-ai/langgraph](https://github.com/langchain-ai/langgraph) (⭐30,965)  
用图结构构建有弹性的语言 Agent。**为什么值得关注**：LangChain 官方出品的 Agent 框架，生产级特性（状态持久、循环、人类干预）。如果要落地多步推理 agent，这是首选库之一。

### CopilotKit：前端 Agent 与生成式 UI 框架
[github.com/CopilotKit/CopilotKit](https://github.com/CopilotKit/CopilotKit) (⭐30,550)  
支持 React + Angular，实现了 AG-UI 协议，让 Agent 能直接操控前端界面。**为什么值得关注**：让你的 AI Agent 不仅能回答问题，还能操作页面上的按钮、输入框，适合做自动化测试或智能客服面板。

### openfang：Agent 操作系统
[github.com/RightNow-AI/openfang](https://github.com/RightNow-AI/openfang) (⭐17,094)  
开源 Agent 操作系统，提供一个运行时环境来管理多个 Agent 实例。**为什么值得关注**：如果你想搭建一个多 Agent 协作平台，这个项目给出完整的 OS 级调度方案，比单纯框架更偏工程落地。

### Open-Generative-AI：开源图像视频生成平台
[github.com/Anil-matcha/Open-Generative-AI](https://github.com/Anil-matcha/Open-Generative-AI) (⭐10,535)  
无审查的开源图像与视频生成工作室，对标 Freepik AI、Krea AI 等。**为什么值得关注**：如果你需要定制图像生成管道，这个项目提供完整的前后端代码，可直接搭建私有生成平台。

### open-design：本地优先的 Claude Design 替代
[github.com/nexu-io/open-design](https://github.com/nexu-io/open-design) (⭐9,609)  
本地优先、开源，替代 Anthropic 的 Claude Design。支持 19 种技能、71 套品牌级设计系统。**为什么值得关注**：设计师和前端开发者可直接在本地运行，无需注册即可生成 UI 组件，适合设计系统快速迭代。
